# 麦麦解析 (NCMai)

## 简介

这是一个收到QQ聊天中的.ncm文件后会自动解码并发送回去的MaiBot插件。主项目是[maibot](https://github.com/Mai-with-u/MaiBot)。

## 配置

有人性化的webui设计，请在webui插件市场安装并在管理页面进行配置:

<br>

### 插件设置
- **开关**：启用/禁用插件。

### 网关设置
- **适配器**：选择文件发送适配器，`snowluma` 或 `napcat`（默认 `snowluma`）。

### 解码设置
- **随机提示语**：解码时随机发送的提示信息（支持 emoji，最多 21 条），每条不超过 10 字（含 emoji）。

### 缓存清理
- **缓存有效期（秒）**：解码后文件保留时间，默认 86400 秒（24 小时），到期自动清理。
- **按需清理**：开启后，每次收到新文件时主动清理已过期的缓存文件。

## 工作原理

1. **触发时机**：插件通过 **BLOCKING Hook** 拦截所有聊天流中的文件消息，检测是否为 `.ncm` 格式。
2. **随机提示**：满足条件后，从配置的提示语列表中随机选取一条发送到当前聊天流。
3. **文件下载**：通过文件 URL 将加密文件下载到本地缓存目录。
4. **格式转换**：将 `.ncm` 文件转换为原始音频格式（自动识别 `.mp3`、`.flac`、`.m4a` 等）。
5. **文件上传**：调用网关的 `upload_private_file` 或 `upload_group_file` API，将解码后的文件上传至 QQ 服务器，获取 `file_id`。
6. **文件发送**：通过 `send.custom` 构造文件消息段，携带 `file_id` 发送回原对话（支持私聊和群聊）。
7. **缓存清理**：解码后的文件保留在临时目录中，超过设定的有效期后自动删除；插件卸载或 MaiBot 重启时也会清理所有缓存。

<br>

## 测试

在webui建立聊天，发送一个.ncm文件触发测试，
如果看到maibot日志显示`[麦麦解析] 解码成功，文件: xxx.mp3`，并收到解码后的音频文件，则说明测试成功。

## 注意

#### 请确保maibot版本为1.0.0或以上,否则webui可能无法正确显示中文插件配置。

#### 本插件依赖 SnowLuma 或 NapCat 提供的 OneBot 网关能力，无需单独配置 OneBot HTTP 地址。

#### 文件发送采用「先上传获取 file_id，再发送文件消息」的标准 OneBot 流程，兼容 SnowLuma 和 NapCat 两种适配器。

#### 解码后的文件保存在系统临时目录，插件卸载或 MaiBot 重启时会自动清理。

#### 请遵守相关法律法规，解码后的文件务必在 24 小时内删除。
